# Файлы данных

## Обязательные файлы

- `settlements.csv` — населенные пункты, координаты, региональная принадлежность
- `regions_bounds.json` — полигоны границ регионов РФ
- `DailyTotalData.csv` — суточные итоги радиации
- `IrradianceData.csv` — почасовая радиация по направлениям
- `SunPosition_Altitude.csv` — высота Солнца
- `SunPosition_Azimuth.csv` — азимут Солнца

## Дополнительные файлы для режима новых данных (ЦФО)

- `cfo_settlements.csv` — населенные пункты ЦФО для нового режима
- `regions_bounds_cfo.json` — границы регионов ЦФО для офлайн-карты в новом режиме
- `MapData/CFO/cfo_manifest.json` — манифест пакета офлайн-карты ЦФО
- `MapData/CFO/cfo_map.mbtiles` — локальная база векторных тайлов/слоев карты ЦФО
- `MapData/CFO/cfo_style.json` — локальный стиль отображения офлайн-карты

Минимальные колонки `cfo_settlements.csv`:

- `region`
- `settlement`
- `latitude`
- `longitude`
- `timezone` (опционально)

Важно: пакет `MapData/CFO` устанавливается один раз вместе с приложением (или data-pack) и не скачивается при каждом запуске.

Подготовка структуры пакета:

- `tools/prepare-cfo-map.ps1` — создает/проверяет структуру `MapData/CFO` и манифест.

## Где должны лежать файлы

Основной путь — рядом с исполняемым файлом (`AppContext.BaseDirectory`).

Дополнительно реализованы fallback-поиски:

1. `Environment.CurrentDirectory`
2. несколько уровней выше от output (для запуска из IDE)

## Кодировки

- Для CSV поддерживаются UTF-8 и Windows-1251 (с автоматическим fallback).
- JSON читается как стандартный UTF-8 текст.

## Признаки проблем с данными

- сообщение "нет данных для региона";
- пустые таблицы в справочнике;
- ошибка загрузки данных при старте.

## Рекомендации

- хранить исходные CSV/JSON в корне проекта;
- не менять заголовки CSV без синхронного обновления парсера;
- проверять корректность десятичных разделителей (`.`/`,`), если меняется источник данных.
- для режима ЦФО указывать атрибуцию OSM в интерфейсе и документации.

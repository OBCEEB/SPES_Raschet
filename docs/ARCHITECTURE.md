# Архитектура проекта

## Общая схема

Приложение построено вокруг трех слоев:

1. UI-слой (WinForms)
2. Слой загрузки/подготовки данных
3. Слой вычислений и визуализации

## Основные модули

- `Form1`  
  Главный экран: карта, справочные таблицы, выбор региона и населенного пункта.

- `SettlementListForm`  
  Диалог выбора населенного пункта внутри выбранного региона.

- `CalculationResultsForm`  
  Выполняет расчет, показывает таблицы и графики, управляет фильтрами направлений.

- `GeoDataHandler`  
  Загружает `settlements.csv` и `regions_bounds.json`; использует устойчивый поиск путей и fallback по кодировке.

- `GeoProcessor`  
  Нормализует названия регионов и сопоставляет данные карты с данными справочника.

- `GeoMapRenderer`  
  Отрисовка карты, преобразование экранных координат в географические, определение региона по клику.

- `DataImporter` (`SolarData.cs`)  
  Загружает и валидирует CSV для радиации/положения Солнца/суточных итогов.

- `MathTools`  
  Линейная интерполяция по ближайшим табличным точкам.

## Поток данных

1. На старте `Form1` вызывает загрузку геоданных и табличных данных.
2. Пользователь кликает по региону на карте.
3. `GeoMapRenderer` определяет регион; `GeoProcessor` находит населенные пункты.
4. Пользователь выбирает населенный пункт в `SettlementListForm`.
5. `CalculationResultsForm` вычисляет интерполированные значения и отображает таблицы/графики.

## Технические решения

- Кодировки CSV: UTF-8 с fallback на Windows-1251.
- JSON: десериализация с `PropertyNameCaseInsensitive=true`.
- Ресурсы данных: копируются в output при сборке.
- Nullable reference types: включены и приведены к чистому состоянию.

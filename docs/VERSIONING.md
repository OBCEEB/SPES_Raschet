# Версионирование

Проект использует **SemVer**:

`MAJOR.MINOR.PATCH`

## Правила изменения версии

- `PATCH` — исправления без изменения внешнего поведения API/функций.
- `MINOR` — новые функции, обратная совместимость сохраняется.
- `MAJOR` — несовместимые изменения или крупная переработка.

## Базовая версия

- `0.1.0` — стартовая стабильная база проекта после приведения к чистой сборке.

## Практика для следующих релизов

- `0.1.1`, `0.1.2` — точечные фиксы.
- `0.2.0` — заметные улучшения функционала.
- `1.0.0` — первый полноценный production-релиз.

## Где хранится версия

Версия задается в `SPES_Raschet.csproj`:

- `Version`
- `AssemblyVersion`
- `FileVersion`
- `InformationalVersion`

При подготовке релиза обновляйте эти значения в одном коммите вместе с `CHANGELOG.md`.

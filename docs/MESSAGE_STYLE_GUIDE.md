# Стандарт пользовательских сообщений

Короткий набор правил для текстов в `MessageBox`, чтобы интерфейс был единообразным и понятным конечному пользователю.

## Общие принципы

- Писать простым языком, без технических терминов уровня реализации.
- Сначала суть проблемы/результата, затем что делать дальше.
- Использовать один стиль заголовков: `СПЭС • <Контекст>`.
- Избегать резких формулировок и лишней детализации стека/внутренних деталей.

## Шаблоны по типам сообщений

| Тип | Иконка | Заголовок | Структура текста |
|---|---|---|---|
| Информация | `Information` | `СПЭС • <Раздел>` | Что произошло успешно + следующий шаг |
| Предупреждение | `Warning` | `СПЭС • <Раздел>` | Что не так + 1-2 действия пользователя |
| Ошибка | `Error` | `СПЭС • <Раздел>` | Что не удалось + подсказка, как исправить |

## Рекомендуемые контексты заголовка

- `СПЭС • Диагностика`
- `СПЭС • Экспорт PDF`
- `СПЭС • Ошибка данных`
- `СПЭС • Данные не найдены`
- `СПЭС • Нужен выбор`

## Примеры

- Информация: `Отчет успешно сохранен.`
- Предупреждение: `Раздел сейчас временно недоступен.`
- Ошибка: `Не удалось прочитать файл данных. Проверьте, что файл находится в папке программы.`

## Чего избегать

- `не реализовано`, `exception`, `stack trace` в пользовательском тексте;
- сокращения без расшифровки (`инфо`, `ошибка CSV` без контекста);
- длинные абзацы без структуры.
